ARCHITECTURE:
The map tab has 4 layers stacked (position absolute):
1. Full-screen MapView (react-native-maps, PROVIDER_GOOGLE, same as SensoryMap.native.tsx)
2. Top overlay: search bar + category chips (horizontal scroll) + radius toggle
3. User location FAB (floating action button, bottom-right)
4. Selected place card (bottom sheet that slides up when a pin is tapped)

DATA FLOW:
1. Get user location with expo-location (same logic as explore.tsx)
2. Fetch nearby places from Google Places API using EXPO_PUBLIC_GOOGLE_MAPS_API_KEY (same as explore.tsx)
3. Batch-fetch sensory scores from Supabase place_reviews table:
   - Query: select place_id, avg(sound_rating), avg(light_rating), avg(crowd_rating), count(*) from place_reviews where place_id IN (fetched place_ids) group by place_id
   - This gives avg sensory scores per place
4. Merge: for each place, attach its sensory scores (or null if no reviews yet)
5. Show pins colored by overall sensory score using scoreColor() from theme

MAP PINS (custom markers):
Each place gets a custom pin with:
- A colored circle (using scoreColor from theme) — green=calm, amber=moderate, red=intense
- The sensory score number inside (or "?" if no data yet)
- The selected pin is larger with a white border ring
- Pins must NOT use react-native-reanimated inside Marker (causes Android crash). Use static Views only.
- Use tracksViewChanges={place.place_id === selectedId} for performance

SELECTED PLACE CARD (bottom sheet, appears when pin tapped):

- Two action buttons: "Get Directions" (opens device maps) + "Full Details" (opens PlaceDetailSheet)
